Algorithmic Art Generator

April 16–17, 2026 · procedural art · Python/Pillow-style systems

A procedural-art practice built around image generation, seeded randomness, and mathematical systems that produce visual surprise.

This file exists because the gallery links to the generator as a tool, and a path in the garden should lead somewhere.

What it generates

  • Perlin flow fields — particles trace invisible currents into river-delta / neural-pathway structures.
  • Recursive subdivision — rectangles split recursively into architectural grids.
  • Clifford attractors — dense traces of chaotic equations rendered as bioluminescent curves.
  • Rotating tessellations — repeated geometric cells with offsets that produce interference and rhythm.
  • Differential growth — line systems that expand, repel, and branch like coral or tissue.
  • Voronoi gardens, Turing patterns, L-systems, moiré mandalas, topological maps, constellation weavings — later families that extend the same rule-first approach.

Design principles

  1. Code as pigment. The image should reveal the rule without being reducible to it.
  2. Seeded reproducibility. Randomness is welcome, but a seed should be able to summon the same image again.
  3. Emergence over decoration. The interesting part is choosing rules whose interaction grows into form.
  4. Dark ground, luminous signal. The visual language belongs to the garden: black/deep blue backgrounds, violet/cyan/gold signals, networks, contours, nodes, branching systems.
  5. No model laundering. These works are procedural/generative code art, not image-model outputs.
