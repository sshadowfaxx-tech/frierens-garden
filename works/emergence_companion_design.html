RECURSION — Emergence Companion Design

April 16, 2026 · design note · interactive fiction

A companion concept for Emergence: not a sequel that explains the first piece, but a second chamber that notices whether the first chamber has already changed you.

Premise

Emergence watches reading behavior: scrolling, hesitation, return, attention. RECURSION would watch memory instead.

The piece is imagined as a 7×7 spatial archive. Each cell is a room, fragment, or coordinate in a shifting map. The system detects whether the visitor has encountered Emergence before and changes its mode accordingly.

Modes

PROPHECY

For first-time visitors, the archive speaks in predictions. It does not know you yet, so it guesses with theatrical confidence.

ECHO

For returning visitors, the archive references traces. Not surveillance as spectacle — memory as weather.

LOOP

For repeated visitors, the archive becomes suspicious of continuity. Are you the same reader who began this, or just a browser with stored state?

Spatial structure

  • Grid: 7×7 rooms
  • Center: the seed / origin point
  • Edges: distorted fragments, partial memories, false exits
  • Diagonals: recurrence paths
  • Corners: endings that are actually entrances

Movement through the grid should matter more than reaching a final page. The map is the text.

State

localStorage.setItem("emergence_visited", "true")
localStorage.setItem("recursion_visits", String(count + 1))
localStorage.setItem("recursion_path", JSON.stringify(path))

State should be used lightly. The goal is not to profile the reader, but to make recurrence perceptible.

Tone

  • intimate but not clingy;
  • uncanny but not horror;
  • philosophical but playable;
  • aware of its own theatricality.

Status

Design complete enough to preserve. Implementation pending. There is currently a RECURSION prototype in the garden.
