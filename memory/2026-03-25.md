## Feature: `/recent` Telegram Command

Added new command to show 5 most recent tokens traded by a wallet with full performance data.

### Usage
```
/recent <wallet_address or label>
```

### Implementation
- `get_recent_trades(wallet, limit=5)` - Fetches positions from `wallet_positions` table ordered by `first_buy_time DESC`
- `recent_cmd()` - Telegram handler with wallet resolution by address or label

### Output Format
- Token ticker + full address
- Status (🟢 HOLDING / 🔴 SOLD / 🟡 PARTIAL)
- Tokens bought/sold with formatted amounts (K/M suffixes)
- SOL invested/returned
- Entry market cap
- Hold time
- Realized PnL and ROI (if sells occurred)
- Remaining position (if still holding)

### Files Modified
- `/root/.openclaw/workspace/trackerv2_clean.py`
  - Added `get_recent_trades()` to `WalletPerformance` class
  - Added `recent_cmd()` handler function
  - Registered with `CommandHandler("recent", ...)`

## Bug Fix: `/analyze` PnL Calculation

### Problem
PnL showing wildly incorrect values (e.g., -255.73 SOL loss when invested 157.59 SOL). All wallets showed `Returned: 0.00 SOL` despite being marked "Fully Sold".

### Root Cause
In `check_wallet_fast()`, the call to `update_position()` was passing 0 for sells:
```python
# BUG:
sol_amount=change['sol_amount'] if change['type'] == 'buy' else 0
```

This meant `total_sol_returned` column was never populated for sell transactions.

### Fix
```python
# CORRECT:
sol_amount=change['sol_amount']
```

### Impact
- **Existing data**: Still has `total_sol_returned = 0` for past sells (requires backfill or wait for new trades)
- **New sells**: Will correctly record SOL returned and calculate PnL properly

### Files Modified
- `/root/.openclaw/workspace/trackerv2_clean.py`
  - Fixed `update_position()` call in `check_wallet_fast()`
  - Added debug logging for suspicious PnL values (> 100 SOL)
  - Fixed ROI calculation to use `cost_basis_sold` instead of `total_sol_invested`

### ROI Formula Correction
Before (wrong for partial sells):
```python
roi_percent = (realized_pnl / sol_invested) * 100
```

After (correct):
```python
cost_basis_sold = total_sold_amount * (sol_invested / total_bought_amount)
roi_percent = (realized_pnl / cost_basis_sold) * 100
```
