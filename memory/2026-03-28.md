### Critical Bugs Fixed in trackerv2_clean.py

#### Bug #1: Broken Round-Robin
The `get_rpc()` method had `self.rpc_index = idx + 1` without modulo wrap-around, causing uneven load distribution across the 6 public RPCs. Fixed to `(idx + 1) % len(self.public_rpcs)`.

#### Bug #2: No RPC Tracking Per Wallet Check
Retry logic could attempt the same failing RPC multiple times for the same wallet check. Added `_get_rpc_excluding()` method that tracks attempted RPCs per wallet check.

```python
attempted_rpcs = set()
for attempt in range(max_retries):
    rpc_url = self._get_rpc_excluding(attempted_rpcs)
    attempted_rpcs.add(rpc_url)
```

#### Bug #3: Limited Error Reporting
Only 429 errors were marking RPCs as failed. Extended `report_rpc_failure()` to handle:
- 429 rate limit → 10 min cooldown
- 5xx server error → 1 min cooldown
- Timeout → 2 min cooldown  
- Connection error → 30 sec cooldown

#### Bug #4: Session Affinity
Transaction fetches were calling `get_rpc()` again, potentially switching RPCs mid-check. Now uses same RPC for both signature fetch and transaction fetch.
