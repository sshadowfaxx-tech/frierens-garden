# Memory Log — 2026-04-08

## Bug Fix: /performance Telegram Parsing Error

**Issue:** `/performance` command returning:
```
❌ Error: Can't parse entities: can't find end of the entity starting at byte offset 1005
```

**Root Cause:** Wallet labels containing underscores (e.g., `whale_wallet_1`) were being interpreted as Markdown italic markers by Telegram's parser. Unpaired underscores caused the "can't find end of entity" error.

**Fix Applied:** Escaped special Markdown characters (`_`, `*`, `[`, `]`) in wallet labels across three functions:

1. `_show_performance_leaderboard()` - Line ~2678
2. `_show_wallet_performance()` - Line ~2593  
3. `_show_wallet_performance_callback()` - Line ~2772

**Code Pattern:**
```python
wallet_label_escaped = wallet_label.replace('_', '\\_').replace('*', '\\*').replace('[', '\\[').replace(']', '\\]')
```

Then used `wallet_label_escaped` instead of `wallet_label` in Markdown-formatted message strings.

**Status:** Fixed in `trackerv2_clean.py` — ready for deployment to FX 6300.
