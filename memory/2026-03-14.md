## Bug Fix: DexCheck PnL Currency (Morning Session — 9:14 AM)

### Problem
Scanner was returning 0 wallets with MEDIUM+ insider score (≥40) despite finding high ROI + high PnL wallets.

### Root Cause
**DexCheck API returns PnL in USD, not SOL.** The filter was checking for `PnL >= 5` but comparing USD values against a SOL threshold, which was far too restrictive.

**Example:** A wallet with $50 USD profit (high) would fail the `>= 5 SOL` filter (~$1000+ USD).

### Solution
Changed PnL filter threshold:
- **Before:** `PnL >= 5 SOL` (expecting ~$1000+ USD)
- **After:** `PnL >= $20 USD` (appropriate for USD values)

2. **Updated display strings** throughout:
   - Filter descriptions now say "$20 USD" instead of "5 SOL"
   - Wallet output shows "$PnL" instead of "SOL" for DexCheck PnL

3. **Cleaned up duplicate code** — removed old `calculate_insider_score()` function that was causing confusion

## DexCheck API Fix: Duration Parameter (Noon Session — 12:26 PM)

### Problem
DexCheck `top-traders-for-pair` endpoint was returning `422 Validation Failed` with error:
```json
{"pair_id":"Field required","duration":"Field required"}
```

Initially attempted to use `"duration": "all"` for lifetime data, but DexCheck API does not support "all" as a valid duration value.

### Root Cause
DexCheck API requires specific duration values. Valid options are:
- `"24h"` — 24 hours
- `"7d"` — 7 days  
- `"30d"` — 30 days
- `"60d"` — 60 days (maximum allowed)

"all" is NOT a valid duration parameter.

### Solution
Changed duration parameter in `fetch_dexcheck_top_traders()`:
- **Before:** `"duration": "all"`
- **After:** `"duration": "60d"`

This provides the maximum available historical window (60 days) for top trader analysis.

### Code Change
```python
params = {
    "chain": "solana",
    "pair_id": pair,
    "duration": "60d"  # Changed from "all" — Options: "24h", "7d", "30d", "60d"
}
```
